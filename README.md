# 小红书养号助手

一个基于Android Accessibility Service的自动化养号工具，通过模拟真实用户行为来提升小红书账号活跃度。

## ⚠️ 重要提示

- **风险提示**：使用自动化工具存在被平台检测和封号的风险
- **建议**：先用小号测试1-2周，确认安全后再用于主号
- **操作量**：建议从每日100篇开始，逐步增加
- **开发者免责**：不承担因使用本工具导致的任何账号损失

## 功能特性

### 核心功能
- ✅ 自动浏览小红书笔记（每日100-200篇）
- ✅ 智能点赞/收藏（根据内容标签动态调整概率）
- ✅ 多账号管理（最多10个账号）
- ✅ 定时任务（每天9:00自动启动）

### 人类化行为模拟
- ✅ 正态分布停留时间（3-20秒，均值8秒）
- ✅ 贝塞尔曲线滑动轨迹
- ✅ 随机误操作模拟（快速滑动后回退）
- ✅ 阅读停顿模拟
- ✅ 随机休息间隔

### 智能策略
- ✅ 目标标签识别（玩偶、毛绒玩具等）
- ✅ 动态互动概率（目标内容3-8%，其他1-3%）
- ✅ 搜索标签功能（每运行3-5次）

## 技术架构

### 项目结构
```
app/src/main/java/com/xhs/auto/
├── data/                    # 数据层
│   ├── model/              # 数据模型
│   │   ├── Account.kt      # 账号模型
│   │   ├── BehaviorConfig.kt  # 行为配置
│   │   └── TaskStatus.kt   # 任务状态
│   ├── AccountManager.kt   # 账号管理
│   └── ConfigManager.kt    # 配置管理
├── automation/             # 自动化核心
│   └── HumanBehaviorSimulator.kt  # 人类行为模拟
├── service/                # 服务层
│   ├── XhsAccessibilityService.kt  # 核心无障碍服务
│   ├── TaskForegroundService.kt    # 前台服务
│   └── TaskAlarmReceiver.kt        # 定时任务
├── ui/                     # 界面层
│   ├── MainActivity.kt     # 主界面
│   ├── AccountActivity.kt  # 账号管理
│   ├── ConfigActivity.kt   # 行为配置
│   └── TaskActivity.kt     # 任务状态
└── XhsApplication.kt       # 应用入口
```

### 关键技术
- **Kotlin Coroutines**：异步任务处理

- **Gson**：数据序列化
- **Material Design**：UI组件

## 使用说明

### 1. 环境要求
- Android 8.0+ (API 26+)
- 小红书App已安装并登录
- 开启无障碍服务权限

### 2. 安装步骤
1. 克隆项目到本地
2. 使用Android Studio打开
3. 同步Gradle依赖
4. 编译并安装到设备

### 3. 首次使用
1. 打开应用，授予必要权限
2. 进入"账号管理"添加账号
3. 进入"行为配置"调整参数（可选）
4. 点击"开始任务"启动

### 4. 配置说明

#### 浏览设置
- **每日浏览目标**：100-200篇（建议从100开始）
- **停留时间**：正态分布，均值8秒，标准差4秒

#### 互动设置
- **目标内容互动概率**：3-8%
- **其他内容互动概率**：1-3%
- **点赞/收藏比例**：70%/30%

#### 标签设置
- **目标标签**：玩偶、毛绒玩具、玩具公仔
- **白名单标签**：手工DIY、礼物推荐

#### 人类化行为
- **贝塞尔曲线滑动**：模拟人类滑动轨迹
- **随机误操作**：5%概率执行误操作
- **阅读停顿**：20%概率停顿2-5秒
